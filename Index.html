Defenitions...

  • Repository:The overall package or project. There is typically a remote of origin repository, and a local copy of it at certain time on your local machine

  • Branch:An extension of a repository. All repositories, or repos, start with a `master` branch. From there, new branches can be created and used for new features. The idea is to keep the master branch clean and working for the final "done" project

  • Commit:A commit is a change to the repository. For every commit made, there is a record of the state of the repository at that given point in time
